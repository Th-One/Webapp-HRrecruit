# คู่มือการใช้งานระบบ HRregister

อัปเดตล่าสุด: 20 กรกฎาคม 2569 (2026-07-20)
Production: https://webapp-hrrecruit.netlify.app

ระบบรับสมัครงานออนไลน์ตามแบบฟอร์ม **F06-005 Rev.2** ของบริษัท มี 2 ส่วนใช้งาน:

| ส่วน | URL | ใช้โดย |
|---|---|---|
| หน้าใบสมัคร | `/` | ผู้สมัครงาน — กรอกข้อมูลเอง |
| หน้า HR | `/hr/` | เจ้าหน้าที่ HR — ดูรายชื่อ ค้นหา ดาวน์โหลด PDF |

## ส่วนที่ 1: สำหรับผู้สมัครงาน

### ขั้นตอนการสมัคร

1. เปิดเว็บไซต์ (หน้าแรก) — จะเห็นแถบ **"ความคืบหน้าในการกรอก"** ด้านบน แสดง % และหมวดที่กรอกแล้ว
2. กรอกข้อมูลตามหมวดต่างๆ (รายละเอียดหมวดด้านล่าง) — ไม่มีหมวดใดบังคับกรอกครบ 100% แต่แนะนำให้กรอกให้ครบเพื่อความสมบูรณ์ของใบสมัคร
3. กด **"👁️ ดูตัวอย่างใบสมัคร"** เพื่อดู PDF ตัวอย่างในแท็บใหม่ก่อนส่งจริงได้ทุกเมื่อ
4. เมื่อกรอกครบแล้ว กด **"บันทึกใบสมัคร"**
5. ระบบจะพาไปหน้า **"ใบสมัครงานพร้อมส่ง HR"** โดยอัตโนมัติ ให้กด:
   - **"📥 บันทึกเป็น PDF ส่ง HR"** เพื่อดาวน์โหลดไฟล์ PDF ใบสมัคร (F06-005 ครบ 2 หน้า)
   - หรือ **"🖨️ พิมพ์"** เพื่อพิมพ์ออกกระดาษโดยตรง

ปุ่ม **"ล้างฟอร์ม"** จะล้างข้อมูลที่กรอกทั้งหมด (มีข้อความยืนยันก่อนล้างเสมอ)

### หมวดข้อมูลในฟอร์ม (เรียงตามลำดับ)

**หน้า 1 ของ F06-005:**

1. **ข้อมูลการสมัคร** — ทราบข่าวจากไหน, ตำแหน่งที่สมัคร, เงินเดือนที่ต้องการ, วันที่พร้อมเริ่มงาน, รายชื่อคนรู้จักใน บ.นีโอคอสเมด (ถ้ามี)
2. **รายละเอียดส่วนตัวผู้สมัคร** — ชื่อ-สกุล (ไทย/อังกฤษ), เชื้อชาติ, สัญชาติ, ศาสนา, วันเกิด, อายุ, น้ำหนัก, ส่วนสูง, เลขบัตรประชาชนและวันออก/หมดอายุบัตร
3. **ที่อยู่ตามทะเบียนบ้าน**
4. **ที่อยู่ปัจจุบัน (ที่ติดต่อได้)** — รวมลักษณะที่พักอาศัย
5. **บิดา / มารดา / พี่น้อง** — ข้อมูลบิดามารดา และตารางพี่น้อง (กดปุ่ม **"+ เพิ่มพี่น้อง"** เพื่อเพิ่มแถวได้)
6. **กรณีติดต่อไม่ได้ — บุคคลที่ติดต่อได้** — ผู้ติดต่อฉุกเฉิน 2 ราย
7. **สุขภาพและประวัติเพิ่มเติม** — สูบบุหรี่, โรคประจำตัว, ประวัติเจ็บป่วยร้ายแรง, ประวัติคดีความ
8. **ความยินยอมให้ตรวจสอบข้อมูลส่วนบุคคล** — ชื่อผู้ลงนามและวันที่ยินยอม (ตาม พ.ร.บ. คุ้มครองข้อมูลส่วนบุคคล)

**หน้า 2 ของ F06-005:**

9. **สถานภาพการสมรสและครอบครัว** — สถานภาพ, คู่สมรส (ถ้ามี), จำนวนบุตร
10. **ความสามารถทางภาษา** — พูด/เขียน/อ่าน/ฟัง ของอังกฤษ จีน ญี่ปุ่น และภาษาอื่น
11. **ความสามารถพิเศษ** — โปรแกรมคอมพิวเตอร์ที่ใช้ได้, ความสามารถอื่นๆ
12. **สถานภาพทางทหาร**
13. **ประวัติการศึกษา** — ตั้งแต่ประถมศึกษาถึงปริญญาโท/อื่นๆ
14. **ความสามารถในการขับขี่ยานพาหนะ** — รถยนต์/รถจักรยานยนต์ และใบขับขี่
15. **รายชื่อบุคคลอ้างอิง** — สูงสุด 2 ราย
16. **ประวัติการทำงาน** — กดปุ่ม **"+ เพิ่มประวัติการทำงาน"** เพิ่มได้สูงสุด 7 รายการ (เรียงจากงานล่าสุดไปเก่าสุด)
17. **งานอดิเรกและกิจกรรม** — งานอดิเรก และสมาชิกภาพสมาคม/ชมรม
18. **การฝึกอบรม ดูงาน ฝึกงาน**

> ทุกตารางที่มีปุ่ม "+ เพิ่ม..." (พี่น้อง, ประวัติการทำงาน, สมาชิกภาพ, การฝึกอบรม) มีปุ่ม **"ลบ"** ในแต่ละแถวสำหรับลบรายการที่เพิ่มผิด

### ข้อควรทราบ

- **การอัปโหลดรูปถ่ายปิดใช้งานชั่วคราว** (เพื่อจำกัดขนาดข้อมูลของระบบ) — กรอกข้อมูลอื่นได้ตามปกติ ไม่มีผลต่อการสมัคร
- รูปแบบวันที่ในฟอร์มเป็น **วว/ดด/ปปปป** (พ.ศ.)
- เมื่อกด "บันทึกใบสมัคร" แล้ว ระบบจะสร้างใบสมัครใหม่ทันที **ไม่สามารถกลับมาแก้ไขจากหน้าเดิมได้** — หากกรอกผิดต้องแจ้ง HR ให้ลบใบเก่าและสมัครใหม่ (ดูหัวข้อ "ลบใบสมัคร" ในส่วนที่ 2)

## ส่วนที่ 2: สำหรับเจ้าหน้าที่ HR

เข้าเว็บไซต์แล้วกดเมนู **"หน้า HR"** หรือไปที่ `/hr/` โดยตรง

### ภาพรวมหน้าจอ

**1. การ์ดสถิติ (บนสุด)** — กดได้ ใช้กรองข้อมูลในกราฟ "ตำแหน่งที่สมัคร" ด้านล่าง

| การ์ด | ผลเมื่อกด |
|---|---|
| ใบสมัครทั้งหมด | กราฟแสดงทุกตำแหน่งจากใบสมัครทั้งหมด |
| เดือนนี้ | กราฟกรองเฉพาะใบสมัครที่ส่งมาในเดือนปัจจุบัน |
| วันนี้ | กราฟกรองเฉพาะใบสมัครที่ส่งมาวันนี้ |

กดการ์ดเดิมซ้ำเพื่อยกเลิกการเลือก (กลับไปแสดงกราฟแบบไม่กรอง)

**2. Dashboard "ตำแหน่งที่สมัคร"** — กราฟแท่งสรุปจำนวนใบสมัครแยกตามตำแหน่งที่สมัคร เรียงจากมากไปน้อย

**3. ช่องค้นหา** — ค้นหาได้จาก: ชื่อ (ไทย/อังกฤษ), ตำแหน่งที่สมัคร, เลขบัตรประชาชน, เบอร์โทร, ชื่อคู่สมรส, ทักษะ, งานอดิเรก, ชื่อบริษัท/ตำแหน่งในประวัติการทำงาน, ชื่อ-เบอร์บุคคลอ้างอิง, ชื่อสถานศึกษา

**4. ตารางรายชื่อใบสมัคร** — คอลัมน์: วันที่บันทึก, ชื่อผู้สมัคร, ตำแหน่งที่สมัคร, บริษัทล่าสุด และการดำเนินการ 3 อย่าง:

- **ดู** — เปิดหน้ารายละเอียดใบสมัคร พร้อมปุ่มดาวน์โหลด PDF / พิมพ์ / ลบ
- **PDF** — ดาวน์โหลดไฟล์ PDF ใบสมัคร (F06-005) โดยตรง ไม่ต้องเปิดหน้ารายละเอียดก่อน
- **ลบ** — ลบใบสมัครออกจากระบบถาวร (มีข้อความยืนยันก่อนลบเสมอ ลบแล้วกู้คืนไม่ได้)

### ข้อควรทราบ

- **ใบสมัครใหม่อาจขึ้นในรายการช้าประมาณ 5–10 วินาที** หลังผู้สมัครกดบันทึก (ข้อมูลถูกบันทึกสำเร็จทันที เพียงแต่รายการภาพรวมอาจตามไม่ทัน) กด refresh หน้าอีกครั้งหากยังไม่เห็น
- การค้นหาและกราฟตำแหน่งทำงานจากข้อมูลที่โหลดไว้ในรายการ — หากเพิ่งมีใบสมัครใหม่เข้ามา ให้ refresh หน้าก่อนค้นหา

## คำถามที่พบบ่อย

**Q: ทำไมอัปโหลดรูปถ่ายไม่ได้?**
A: ปิดใช้งานชั่วคราวเพื่อควบคุมขนาดข้อมูลของระบบ ไม่กระทบข้อมูลอื่นในใบสมัคร ระบบจะเปิดให้ใช้อีกครั้งเมื่อพร้อม

**Q: กรอกฟอร์มผิด แก้ไขได้ไหม?**
A: หน้าใบสมัครไม่มีปุ่มแก้ไขย้อนหลัง หากกรอกผิดพลาด ให้แจ้ง HR ลบใบสมัครเดิมออก (ปุ่ม "ลบ" ในหน้า HR) แล้วให้ผู้สมัครกรอกใหม่

**Q: ส่งใบสมัครไปแล้วแต่ไม่เห็นในหน้า HR**
A: รอสักครู่ (5–10 วินาที) แล้วกด refresh หน้า หากยังไม่ขึ้นให้ตรวจสอบว่าได้กด "บันทึกใบสมัคร" และเห็นข้อความ "บันทึกใบสมัครสำเร็จ" แล้วหรือไม่

**Q: PDF ที่ได้คือแบบฟอร์มอะไร**
A: แบบฟอร์ม F06-005 Rev.2 ของบริษัท (2 หน้า) ระบบนำข้อมูลที่กรอกไปวางลงบนแบบฟอร์มต้นฉบับให้อัตโนมัติ

## ภาคผนวก: ข้อมูลสำหรับผู้ดูแลระบบ

รายละเอียดเชิงเทคนิค (สถาปัตยกรรม, การ deploy, โครงสร้างข้อมูล) ไม่ได้อยู่ในคู่มือฉบับผู้ใช้งานนี้ — ดูได้จาก:

- `README` และคอมเมนต์ในซอร์สโค้ด (`server.js`, `storage.js`, `pdf/generate-f06.js`)
- ประวัติ commit ใน git log สำหรับเหตุผลของการเปลี่ยนแปลงแต่ละครั้ง
- `scripts/test-storage.js` สำหรับทดสอบชั้นเก็บข้อมูลก่อน deploy

สรุปสั้นสำหรับผู้ดูแลระบบ:

- **Stack**: Node.js + Express, สร้าง PDF ด้วย `pdf-lib`/`fontkit`, โฮสต์บน Netlify Functions
- **ที่เก็บข้อมูล**: Netlify Blobs — แต่ละใบสมัครเก็บแยกเป็น blob ของตัวเอง (`app/{id}`) พร้อม index สรุปย่อสำหรับหน้ารายการ
- **การ deploy**: ใช้ `netlify deploy` สร้าง draft แล้ว publish ผ่าน Netlify API (`restoreSiteDeploy`) — คำสั่ง `netlify deploy --prod` ของ CLI ใช้ไม่ได้ในสภาพแวดล้อมนี้ (คืน `Forbidden`)
- **การอัปโหลดรูป**: ปิดด้วยสวิตช์ `PHOTO_UPLOAD_ENABLED` ทั้งฝั่ง client (`public/js/form.js`) และ server (`server.js`) — ต้องเปิดทั้งสองจุดพร้อมกัน
